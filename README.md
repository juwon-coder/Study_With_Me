# StudyBuddy - 스터디 매칭 홈페이지

## 1. 프로젝트 개요

이 프로젝트는 그룹 스터디를 위한 스터디 매칭 홈페이지인 "StudyBuddy"를 구현한 것입니다. 사용자는 이 플랫폼을 통해 자신에게 맞는 스터디 그룹을 검색하고 참여하거나, 직접 새로운 스터디를 개설할 수 있습니다. 본 프로젝트는 [PRD 문서](#prd-문서)에 명시된 요구사항을 기반으로 개발되었습니다.

## 2. Assignment 요구사항

- **PRD 문서**: 제품 요구사항 정의서(Product Requirements Document)를 작성하여 제품의 목표, 기능, 기술 스택 등을 정의했습니다.
- **웹사이트 구현**: PRD를 바탕으로 HTML, CSS, JavaScript를 사용하여 간단한 웹사이트를 구현했습니다. 최소 3개 페이지(홈, 소개, 문의)를 포함하고, JavaScript를 이용한 상호작용 기능(스터디 개설, 검색, 상세 보기, 참여 신청 등)을 구현했습니다.

## 3. 기능 목록

### 3.1. 사용자 기능
- **스터디 검색**: 키워드, 카테고리, 진행 방식, 모집 상태별 필터링을 통해 원하는 스터디를 찾을 수 있습니다.
- **스터디 상세 보기**: 스터디의 상세 정보(설명, 모집 인원, 일정 등)를 확인하고 참여 신청을 할 수 있습니다.
- **스터디 참여 신청**: 관심 있는 스터디에 참여 신청을 할 수 있습니다.

### 3.2. 스터디 리더(개설자) 기능
- **스터디 개설**: 새로운 스터디 그룹을 개설하고 상세 정보를 입력할 수 있습니다.

## 4. 기술 스택

- **프론트엔드**: HTML5, CSS3, JavaScript (Vanilla JS)
- **데이터 저장**: 웹 브라우저의 `localStorage`를 사용하여 임시 데이터를 관리합니다. (백엔드 서버 없이 클라이언트 측에서만 데이터를 처리)

## 5. 프로젝트 실행 방법

1.  **파일 다운로드**: 프로젝트의 모든 파일을 로컬 환경으로 다운로드합니다.
2.  **브라우저에서 열기**: `index.html` 파일을 웹 브라우저(Google Chrome, Firefox 등)로 엽니다.
    *   `index.html`: 메인 페이지
    *   `about.html`: 프로젝트 소개 페이지
    *   `contact.html`: 문의 페이지
    *   `create-study.html`: 스터디 개설 페이지
    *   `study-list.html`: 모든 스터디 목록 페이지
    *   `study-detail.html`: 스터디 상세 정보 페이지

## 6. 파일 구조

```
.  
├── index.html         // 메인 페이지
├── about.html         // 소개 페이지
├── contact.html       // 문의 페이지
├── create-study.html  // 스터디 개설 페이지
├── study-list.html    // 스터디 목록 페이지
├── study-detail.html  // 스터디 상세 페이지
├── style.css          // 전역 스타일 시트
└── script.js          // 전역 JavaScript 로직
```

## 7. PRD 문서

본 프로젝트의 상세 요구사항은 별도의 PDF 문서로 제출됩니다.
