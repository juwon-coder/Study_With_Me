# Study_with_Me - 스터디 매칭 홈페이지

## 1. 프로젝트 개요

*   **제품명:** Study_with_Me (스터디 매칭 홈페이지)
*   **타겟 고객:** 학습 그룹을 찾거나 스터디를 개설하려는 학생, 직장인 및 일반인
*   **핵심 목적:** 사용자가 자신의 학습 목표와 관심사에 맞는 스터디 그룹을 쉽게 찾고 참여할 수 있도록 돕는 온라인 플랫폼을 제공합니다. 스터디 리더는 효율적으로 그룹을 개설하고 관리할 수 있으며, 사용자는 자신이 개설하거나 신청한 스터디를 한눈에 확인할 수 있습니다. 모든 기능은 클라이언트 측에서만 작동하며, 데이터는 웹 브라우저의 로컬 스토리지에 임시 저장됩니다.

## 2. 프로젝트 목표

*   사용자 친화적인 인터페이스를 통해 스터디 그룹 검색 및 참여 과정을 단순화합니다.
*   다양한 스터디 카테고리, 검색 및 필터링 옵션을 제공하여 사용자 맞춤형 스터디 매칭률을 높입니다.
*   스터디 그룹 개설 및 관리에 필요한 기본적인 클라이언트 측 기능을 제공하여 운영 효율성을 증대합니다.
*   사용자가 자신이 개설하거나 신청한 스터디를 쉽게 관리할 수 있는 기능을 제공합니다.
*   안정적이고 반응형이며, 기본 웹 기술(HTML, CSS, JavaScript)만으로 구현된 웹 기반 서비스를 구축합니다.

## 3. 기능 목록

### 3.1. 사용자 공통 기능

*   **스터디 검색 (Must-have):**
    *   키워드 검색 (스터디명, 주제, 리더명)
    *   카테고리 필터링 (어학, 코딩, 자격증, 취업, 학업, 취미/교양, 기타)
    *   진행 방식 필터링 (온라인, 오프라인, 혼합)
    *   모집 상태 필터링 (모집 중, 모집 완료)
    *   **내가 개설한 스터디 필터링:** 현재 브라우저에서 개설한 리더명으로 필터링하여 자신의 스터디만 볼 수 있습니다.
*   **스터디 목록 보기 (Must-have):**
    *   모든 개설된 스터디를 카드 형태로 목록에 표시합니다.
    *   각 카드에는 스터디명, 카테고리, 리더명, 모집 현황, 간략한 설명, 모집 상태가 표시됩니다.
*   **스터디 상세 보기 (Must-have):**
    *   선택한 스터디의 상세 정보 (스터디명, 카테고리, 리더명, 모집 인원, 현재 신청 인원, 진행 방식, 예상 일정, 모집 마감일, 상세 설명)를 표시합니다.
    *   해당 스터디의 모집 현황 (현재 인원/총 인원)을 명확하게 보여줍니다.
*   **사이트 소개 (Must-have):**
    *   Study_with_Me의 목적, 주요 특징, 사용 방법, 미래 계획 등을 설명하는 페이지를 제공합니다. 시각적 가독성을 높인 디자인이 적용되어 있습니다.
*   **문의하기 (Must-have):**
    *   이름, 이메일, 메시지를 입력하여 문의할 수 있는 폼을 제공합니다. (실제 전송 기능 없이 클라이언트 측에서 메시지 확인)

### 3.2. 스터디 리더 기능 (스터디 개설자, 현재 브라우저 기준)

*   **스터디 개설 (Must-have):**
    *   스터디 리더명, 스터디명, 카테고리, 설명, 총 모집 인원, 진행 방식 (온/오프라인), 예상 스터디 일정, 모집 마감일 등을 입력하여 새로운 스터디를 개설할 수 있습니다.
    *   스터디 ID는 자동으로 고유하게 생성됩니다.
    *   입력된 스터디 리더명은 현재 브라우저에서 "내가 개설한 스터디"를 식별하는 데 사용됩니다.
*   **스터디 수정 (Must-have):**
    *   스터디 상세 페이지에서 "내가 개설한 스터디"일 경우 '수정하기' 버튼이 표시됩니다.
    *   '수정하기' 버튼 클릭 시, 개설 폼에 기존 스터디 정보가 미리 채워지고, 리더명을 제외한 모든 필드를 수정할 수 있습니다.
    *   수정된 내용은 로컬 스토리지에 반영됩니다.
*   **스터디 삭제 (Must-have):**
    *   스터디 상세 페이지에서 "내가 개설한 스터디"일 경우 '삭제하기' 버튼이 표시됩니다.
    *   '삭제하기' 버튼 클릭 시, 확인 절차를 거쳐 스터디가 로컬 스토리지에서 제거되고 "내 스터디" 목록으로 이동합니다.
*   **내 스터디 보기 (Should-have):**
    *   '내 스터디' 페이지에서 "내가 개설한 스터디" 탭을 통해 현재 브라우저에서 개설한 모든 스터디 목록을 볼 수 있습니다.
    *   각 스터디 카드에는 스터디명, 카테고리, 리더명, 모집 현황, 간략한 설명, 모집 상태가 표시됩니다.

### 3.3. 스터디 참여자 기능 (스터디 신청자, 현재 브라우저 기준)

*   **스터디 참여 신청 (Must-have):**
    *   "내가 개설한 스터디"가 아닌 스터디 상세 페이지에서 '스터디 참여 신청' 버튼이 표시됩니다.
    *   버튼 클릭 시, 스터디의 현재 신청 인원(`currentMembersCount`)이 1 증가하고, 신청자 목록(`appliedMembers`)에 `currentLeader` (가상 사용자)가 추가됩니다.
    *   모집 인원이 가득 차면 스터디 상태가 '모집 완료'로 자동 변경되고 신청 버튼이 비활성화됩니다.
    *   한 번 신청하면 해당 브라우저에서는 '신청 완료'로 버튼이 변경되고 비활성화되어 재신청할 수 없습니다.
*   **내 스터디 보기 (Should-have):**
    *   '내 스터디' 페이지에서 "내가 신청한 스터디" 탭을 통해 현재 브라우저에서 신청한 모든 스터디 목록을 볼 수 있습니다.
    *   각 스터디 카드에는 스터디명, 카테고리, 리더명, 모집 현황, 간략한 설명, 모집 상태가 표시됩니다.

## 4. 기술 및 디자인 요구사항

*   **4.1. 기술 스택:**
    *   **프론트엔드:** HTML5, CSS3, JavaScript (Vanilla JS)
    *   **데이터 저장:** 웹 브라우저의 `localStorage`를 활용하여 스터디 정보 및 사용자 관련 상태(내가 개설한 리더명 목록, 마지막으로 개설/신청한 리더명 등)를 임시 관리합니다.
*   **4.2. UI/UX 가이드라인:**
    *   **반응형 디자인:** 다양한 디바이스(데스크탑, 태블릿, 모바일)에서 최적화된 사용자 경험을 제공합니다.
    *   **직관적인 내비게이션:** 사용자가 원하는 정보에 쉽게 접근할 수 있도록 명확한 메뉴 구조(홈, 소개, 문의, 스터디 개설, 내 스터디)를 가집니다.
    *   **심플하고 깔끔한 디자인:** 시각적 복잡성을 줄이고 콘텐츠에 집중할 수 있도록 디자인되었습니다. `about.html` 페이지는 시각적 가독성을 높이기 위해 그리드 배치 등의 디자인 요소를 포함합니다.
    *   **접근성:** 기본적인 웹 접근성 표준을 준수합니다.
    *   **버튼 일관성:** 스터디 상세 페이지의 '수정하기', '삭제하기', '참여 신청' 버튼의 크기와 스타일이 일관성을 유지합니다.
*   **4.3. 통합 요구사항:**
    *   없음 (클라이언트 측 단독 웹 애플리케이션으로 구축, 백엔드 연동 없음)

## 5. 프로젝트 실행 방법

1.  **파일 다운로드**: 프로젝트의 모든 파일을 로컬 환경으로 다운로드합니다.
2.  **브라우저에서 열기**: `index.html` 파일을 웹 브라우저(Google Chrome, Firefox 등)로 엽니다.
    *   `index.html`: 메인 페이지
    *   `about.html`: 프로젝트 소개 페이지
    *   `contact.html`: 문의 페이지
    *   `create-study.html`: 스터디 개설 페이지
    *   `study-list.html`: 모든 스터디 목록 페이지
    *   `study-detail.html`: 스터디 상세 정보 페이지
    *   `my-studies.html`: 내가 개설하거나 신청한 스터디 목록 페이지
3.  **데이터 초기화 (선택 사항):** `localStorage`에 기존 데이터가 남아있을 경우, 브라우저 개발자 도구(F12)의 `Application` 탭 -> `Local Storage`에서 `studies` 및 `myCreatedLeaders` 키를 삭제하여 초기 예시 스터디로 다시 시작할 수 있습니다.

